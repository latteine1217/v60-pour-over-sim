Project Showcase
Why This Repo Exists
A pour-over simulator that treats flow, temperature, and extraction as one coupled physical system.
Most coffee tools stop at recipe calculators or static brew ratios. This project instead tracks the bed as a changing porous medium and now ties that model back to measurement: real-time poured volume, drained volume, measured PSD, ceramic dripper heat capacity, and final cup temperature all constrain the same state trajectory.
Measured PSD The grinder is no longer represented by an idealized fractal PSD. Kinu 29 measured bins now drive D10, fines indicators, surface area, shell accessibility, and diffusion path.
Split Clogging Bed resistance now separates early throat blocking from later deposition, so `k_beta` is no longer one opaque lumped decay coefficient.
Bin-Resolved Extraction Each PSD bin carries its own `A_i`, `L_i`, `M_i`, and dissolved concentration, then recombines into the cup-side fast/slow output.

01   Flow Story
Hydraulic Mechanism
The brew is controlled by pressure head, bed resistance, and late-opening bypass.
This section is the hydraulic backbone of the model. The state does not assume one fixed flow path: water can go through the bed, be blocked by capillary and gas thresholds, or divert into wall-channel bypass once free-water head becomes large enough. The bed-resistance term is now explicitly split into early throat clogging and later deposition.
Drive `h(t)` and `Q_in` raise the pressure head. This is the immediate term that pushes water through the porous bed.
Resistance `k_eff` decays with fines migration, swelling, and compression, so the same recipe can move into a slower hydraulic regime.
Threshold `h_cap` and `h_gas(t)` subtract from usable head, which is why early brew flow can stay suppressed even with visible water above the bed.
Leakage Path `Q_bp` stays small at low head and opens later, so bypass is modeled as a head-activated parallel pathway rather than a constant loss.
Darcy extraction flow
Q_ext ∝ k_eff · h_eff / μ(T)
h_eff = h − h_cap − h_gas(t) + h_cap,wet
Split fines clogging
k_eff = k₀ / [ (1 + A_throat · (1 − e−V_out/V_char)) · (1 + β_dep · V_out) ]
early throat blocking saturates; later deposition keeps accumulating
CO₂ back-pressure decay
h_gas(t) = h_gas₀ · exp(−t / τCO₂)
τ_CO₂ ≈ 35 s　（Cameron et al. 2020）
Bypass flow (rib channels)
Q_bp = ψ · h_free · shape(h_free) · activation(h_free)
low water head ≈ no bypass; higher free-water head opens rib-channel flow
Particle swelling — Kozeny-Carman
φ(sat) = φ₀ − Δφ · sat
k ∝ φ³ / (1−φ)²　pore connectivity
In the generated diagnostics, this logic resolves into measurable outputs: water-head collapse, bypass spikes, permeability retention, and the point where the bed becomes fully wetted.
02   Extraction Story
Shrinking-core accessibility (per bin)
C_eff,i = C_sat(T) · (Mi / M₀,i)β, β = 1.5
Bin-resolved Noyes-Whitney transport
k_ext,i(T) = η · A_i · D(T) / L_i · exp(−Eₐ / RT)
Bed concentration ODE — bin-resolved CSTR
dCi/dt = disi / V_liq − Ci · Q_ext / V_liq
Solid-phase depletion
dMi/dt = −disi

Fast (Acids / Sweetness) Eₐ = 15 kJ/mol

03   System Map
Model Architecture
One measured pipeline, one coupled engine, several projection surfaces.
This is the repo's main structure, not a decorative dependency chart. Measured data enters through a strict I/O layer, physically interpretable closures are assembled in params.py, the full brew evolves inside one coupled ODE in core.py, and every fit, benchmark, and figure is downstream of that same state trajectory.
Input Layer Measured flow, thermal CSV, PSD bins, roast profile, dripper hardware, and ambient metadata are all fixed upstream inputs, not silent optimizer degrees of freedom.
Core Layer `h, V_out, V_bed, V_poured, V_liq, sat, T, T_effluent, T_dripper, chi_struct, xi_pref, {C/M bins × axial layers}` live inside one state update loop.
Projection Layer Observation lag, server mixing, fitting, benchmark gating, and exported figures are separate surfaces laid on top of the same physical simulation.
Read the graph from left to right. Nothing on the page should bypass the measured-input layer or invent a second hidden model: closures feed one ODE engine, the observation layer translates that state into what the brewer measures, and fitting / benchmark / visualization are downstream technical surfaces rather than alternative physics.
This is why the repo can stay physically interpretable while still being practical: each node has one job, but the state trajectory is shared. Hydraulic fixes, thermal semantics repairs, and PSD updates all meet in the same solver instead of forking into separate pipelines.
04   Temperature Regime
Two-node thermal model
dT/dt = (Q_in/V_eff)(T_brew − T) − λ_cool(T − T_amb) − λ_ld(T − T_dripper)
dT_dripper/dt = λ_dl(T − T_dripper) − λ_da(T_dripper − T_amb)
Dynamic viscosity (water)
μ(T) = μ_ref · μ_rel(T) / μ_rel(T_ref)
μ_rel(T): empirical fit to the user-provided viscosity curve
Effect on flow rate
Q ∝ k / μ(T)
Lower T → higher μ → slower Q → longer contact time
Saturation concentration (Henry-type)
C_sat(T) = C_sat_ref · exp[α · (T − T_ref)]
Higher T → more soluble → faster EY build-up
The landing-page view here is not “temperature is good or bad”; it is that temperature changes both throughput and chemistry, and the slow bitter fraction reacts more strongly than the fast bright fraction.

Capillary pressure cutoff
Q → Q · σ_cap(h)

drip-filter transition: h < h_cap → Q → 0
Bloom saturation — cubic Hermite
sat = V_poured / V_absorb(dose)
smooth_step(sat, 0, 1) = 3s² − 2s³ [C¹]
V_absorb_dry = 0.5 mL/g (CO₂-corrected)
V_absorb_full = 1.64 mL/g (medium roast)
Pressure-head closure
h_eff = raw_head · sigmoid(raw_head / ε)
smooth positive-part closure near the capillary / gas threshold
prevents numerical chatter at raw_head ≈ 0

Technical Highlights
Model Surface
The repo exposes a reduced-order but fully coupled brew model, not a static recipe calculator.
The technical point of this project is that the brew is treated as a stateful system. Hydraulic resistance, thermal loss, accessibility, and cup-side dilution all evolve together. The current repo no longer maintains an older fractal-PSD branch; the measured-bin model is now the main line.
State Vector `h, V_out, V_poured, sat, {C_fast,i, M_fast,i, C_slow,i, M_slow,i}, T, T_dripper, chi_struct`
Primary Outputs Brew time, drain time, bypass ratio, permeability retention, TDS, in-cup EY, dissolved EY, measured D10 diagnostics, and fast/slow cup balance.
Physical Closures Darcy flow, split clogging, capillary cutoff, wet-bed capillary boost, two-node thermal coupling, roast-dependent chemistry, and bin-resolved Noyes-Whitney transport.
Core State
state = [h, V_out, V_poured, sat, {C_fast,i, M_fast,i, C_slow,i, M_slow,i}, T, T_dripper, chi_struct]
Hydraulic Output Layer
Q_ext, Q_bp, k_eff, sat, h, brew_time, drain_time, throat/deposition clog diagnostics
Cup Output Layer
C_out, TDS, EY_cup, EY_dissolved, EY_fast, EY_slow, cup temperature fit error
Project scope, installation, and reproducibility are intentionally kept in the README. This page is for surfacing the current best model, the measured calibration, and the regime figures it produces.
